# Pluvii Design Methodology

## Purpose

This document defines the design and content methodology for the Pluvii website. It is the reference for new pages, components, content types, campaigns, and future features so the site can grow without drifting in style or structure.

The goal is not to freeze the design. The goal is to make decisions repeatable.

## Brand Direction

### Core feeling

Pluvii should feel:

- approachable
- easy to understand
- playful
- friendly
- bold
- optimistic
- practical

The site should carry a slight "we are excited to save the planet" energy, but it should not drift into vague, mystical, or overly hippie aesthetics. It should feel grounded in land, water, and real work.

### Brand personality

Pluvii sits between three modes:

- credible ecological practice
- practical land guidance
- clear, contemporary communication

If a design choice feels too corporate, too sterile, or too spiritual, it is probably off-center. The right answer usually has warmth, clarity, and confidence.

### Tone of voice

Content should sound:

- concrete, not abstract
- informed, not academic
- hopeful, not naive
- warm, not casual to the point of being sloppy
- clear, not jargon-heavy

Prefer direct explanations, specific examples, and tangible outcomes.

## Visual Principles

### Visual stance

The visual system should combine:

- clean graphics
- structured layouts
- real-world imagery
- field photos and videos
- satellite or map-based visuals

This should create a blend of editorial clarity and landscape reality.

### Design rules

- Use strong composition and generous whitespace.
- Let pages feel airy and legible rather than dense.
- Pair bold sections with soft edges where useful.
- Avoid generic startup visuals and stock sustainability tropes.
- Avoid clutter, ornamental eco graphics, and visual noise.
- When in doubt, simplify the layout and strengthen the hierarchy.

### Shape language

Preferred forms:

- rounded rectangles
- large soft panels
- clean line dividers
- bold section blocks
- simple geometric accents

These shapes should suggest water, terrain, and friendliness without becoming childish.

## Color System

### Core palette

The main palette should be built from:

- light blue
- light green
- white
- black

The default feel should be light, fresh, and optimistic.

### Recommended palette foundation

These values are a starting point, not a rigid final brand lockup:

- Water Blue: `#79D8F3`
- Soft Blue: `#C0F1FF`
- Leaf Green: `#A4E38E`
- Soft Green: `#D2F3C6`
- White: `#FFFFFF`
- Near Black: `#1E1E1E`
- Mist Gray: `#DCDEDE`

### Usage guidance

- Use white or very pale tints as the primary canvas.
- Use black or near-black for text and key contrast.
- Use light blue as the lead accent in interfaces and atmospheric backgrounds.
- Use light green as the secondary accent to reinforce regeneration and land.
- Use stronger contrast sparingly for calls to action, overlays, and navigation states.
- Keep the palette bright and calm rather than earthy and muddy.

### Color balance

- Blue should represent water, openness, and clarity.
- Green should represent land, renewal, and growth.
- White should keep the system breathable.
- Black should provide confidence and readability.

If a page begins to feel too cold, introduce more green. If it begins to feel too soft or vague, strengthen black contrast and structural lines.

## Typography

### Primary typeface

The main brand typeface should be [Inter](https://rsms.me/inter/).

Inter should be used for:

- logo wordmark
- navigation
- headlines
- body text
- buttons
- labels
- UI components

### Typographic character

The typography should feel:

- modern
- clean
- legible
- neutral with warmth from layout and color rather than decorative letterforms

The site should not depend on rustic or overly expressive typography to signal ecology. The ecological character should come from content, imagery, palette, and composition.

### Type rules

- Use a clear size hierarchy with strong headline contrast.
- Keep body text highly readable and comfortably spaced.
- Prefer sentence case over all caps, except for rare utility labels.
- Avoid using too many font weights in one screen.
- Use bold weight intentionally for emphasis, not everywhere.

## Logo Methodology

### Current direction

The current placeholder logo suggests a simple monochrome symbol with a blocky, geometric quality. Treat this as an early shape language reference rather than a final system.

### Logo system recommendation

Pluvii should eventually have:

- a wordmark set in Inter or a modified Inter-based treatment
- a standalone symbol that can work at small sizes
- a combined lockup for headers, social avatars, and print or presentation use

### Logo qualities

The symbol should feel:

- simple
- memorable
- geometric
- friendly
- scalable

It should avoid:

- overly intricate natural motifs
- generic leaf-only symbolism
- mystical or spiritual symbolism
- visual clichés from sustainability branding

### Logo themes worth exploring

- water flow
- contour lines
- catchment shapes
- droplets abstracted into geometry
- terrain blocks
- modular landscape forms

The best mark will likely be abstract enough to feel modern, but grounded enough to suggest land and water systems.

## Image And Media Direction

### Media mix

Pluvii should deliberately mix:

- field photography
- satellite imagery
- maps
- diagrams
- short video loops
- occasional clean illustrations

The site should not rely on only one of these.

### Photography rules

- Prefer real landscapes over generic lifestyle images.
- Show land, water, topography, vegetation, and intervention context.
- Favor images with a sense of place and scale.
- Use people sparingly and naturally.
- Avoid staged team photos unless there is a strong reason.

### Satellite and mapping visuals

- Use satellite or map imagery where it adds understanding, not decoration.
- Pair technical views with explanatory captions.
- Keep overlays simple and legible.
- Use them to show before-and-after logic, land reading, or intervention strategy.

### Graphics and diagrams

- Diagrams should simplify, not impress.
- Use a small set of visual conventions for arrows, highlights, labels, and overlays.
- Make technical information feel inviting.

## Layout And Component Principles

### Layout rules

- Build pages as clear sections with a strong reading rhythm.
- Alternate between text-led and media-led sections.
- Keep widths readable and avoid long line lengths.
- Use visual contrast between sections through background, imagery, or panel treatment.

### Component behavior

Every component should answer one of these jobs:

- explain
- guide
- prove
- invite action

If a component does not clearly do one of those jobs, it probably does not need to exist.

### Interaction style

- Interactions should feel calm and responsive.
- Motion should support understanding, not novelty.
- Use subtle transitions and reveal patterns.
- Avoid busy animation or trendy interface effects.

## Page Archetypes

### Homepage

The homepage should:

- explain what Pluvii does in one clear statement
- make the offer feel approachable
- show that the work is real and grounded
- point visitors toward contact, projects, or educational content

The homepage is not a dumping ground. It should be selective and directional.

### Blog posts

Blog posts exist to educate, build trust, and document thinking.

Each post should usually do one of these:

- explain a land or water concept clearly
- document field observations
- unpack a method or decision
- answer a practical question
- share lessons from a project

Blog posts should favor:

- clear headlines
- strong introductions
- useful subheads
- diagrams or images when they improve understanding
- practical takeaways

Avoid vague thought leadership content.

### Project pages

Project pages should function as case studies or field examples.

Each project page should aim to include:

- context
- landscape or site conditions
- challenge
- analysis approach
- intervention logic
- expected or observed outcomes
- visuals

Project pages should be evidence-oriented, even when a project is early or incomplete.

### Landing pages

Landing pages may be created for:

- a service offer
- a specific audience
- a region
- a partner collaboration
- a campaign or outreach push

Landing pages should still look and feel like Pluvii. They can flex in emphasis, but not in brand logic.

Each landing page should have:

- one primary audience
- one primary message
- one primary call to action
- one proof layer

### Utility and support pages

Pages like contact, about, FAQ, and future resource pages should prioritize clarity over brand theatrics. They should still feel designed, but they should be frictionless to use.

## Editorial Methodology

### Content strategy

The site should combine:

- explanatory education
- proof of practice
- trust building
- conversion support

That means content should not separate "marketing" and "knowledge" too aggressively. The strongest content will often do both.

### Writing rules

- Start with the reader's question or need.
- State things plainly.
- Use concrete nouns and verbs.
- Prefer examples from land, water, and site conditions.
- Define specialist terms quickly when needed.
- End with useful next steps or a clear takeaway.

### Content quality filter

Before publishing, ask:

- Is this understandable to a non-expert landowner?
- Does this add practical value?
- Does this sound like Pluvii rather than generic sustainability copy?
- Does this help a visitor trust the work?
- Does this page have a clear next step?

## Scaling Rules

### When adding new features

Any new feature should align with these checks:

- Does it support the core mission of understanding land and designing water retention?
- Does it fit the approachable and bold visual language?
- Does it preserve clarity?
- Can it work in a static-first architecture?
- Can it support multilingual growth later?

If the answer is no to several of these, reconsider the feature.

### When adding new pages

New pages should not invent their own design logic. They should inherit:

- the same palette
- the same typography
- the same spacing rhythm
- the same tone of voice
- the same component vocabulary

Variation should come from content and section composition, not from random visual reinvention.

### When introducing new components

Only add a new component when:

- an existing one cannot reasonably serve the need
- the content pattern is likely to repeat
- the new component is simple enough to maintain

Prefer a small, robust component library over a large, fragile one.

## Non-Goals

Pluvii should not become:

- a generic tech startup site
- a rustic eco craft brand
- a mystical wellness project
- a dense academic portal
- a visual collage with no hierarchy

## Working Default

Until the system evolves further, the default Pluvii design answer is:

"Use Inter, keep the page bright and breathable, combine light blue and light green with strong black structure, show real land whenever possible, explain things clearly, and make optimism feel practical."
